2

CLAUDE.md로 맥락 심어주기

하네스 · 맥락
왜 이렇게 할까요? 매번 채팅으로 같은 설명을 반복하는 대신, CLAUDE.md라는 파일 하나에 프로젝트의 배경과 규칙을 적어두면 AI가 세션을 새로 시작해도 그 내용을 계속 기억해요. 이런 파일에는 두 단계가 있어요 — 모든 프로젝트에 공통으로 적용되는 전역 설정과, 이 프로젝트에만 적용되는 프로젝트 설정.
~/.claude/CLAUDE.md ← 전역 (모든 프로젝트 공통 규칙) ./CLAUDE.md ← 프로젝트 (이 사이트만의 규칙)
이 프로젝트 루트에 CLAUDE.md를 만들어달라고 부탁하세요
Tip. CLAUDE.md는 짧게 유지하는 게 좋아요 — 매 세션마다 통째로 읽히기 때문에, 길어지면 그만큼 낭비예요. 필요한 자세한 설명은 나중에 배울 Skill로 분리해요.
✅ 확인해보세요
프로젝트 폴더 안에 CLAUDE.md 파일이 실제로 생겼나요? (ls 또는 파일탐색기로 확인)
3

Plan Mode로 설계부터 하기

하네스 · 계획
왜 이렇게 할까요? 바로 "만들어줘"라고 하면 AI가 여러분 머릿속 전제를 다 담지 못한 채로 짐작해서 진행해요. Plan Mode로 들어가면 AI가 코드를 쓰기 전에 먼저 질문하고, 계획을 세운 뒤 승인을 받고 나서야 실행해요 — 시간은 조금 더 걸리지만 완성도가 훨씬 높아져요.
터미널에서 Plan Mode로 들어간 뒤 (Shift+Tab 또는 /plan 입력) 이렇게 부탁하세요
사람이 할 일. AI가 던지는 질문에 하나씩 답하고, 마지막에 나온 계획을 읽어본 뒤 문제없으면 승인(Yes)하세요.
✅ 확인해보세요
index.html / style.css / script.js 같은 파일들이 만들어졌나요?

기능 2 — 위 기능이 잘 되면, 그다음에

5

Hook으로 안전장치 만들기

하네스 · 구조 심화
왜 이렇게 할까요? 지금까지 만든 규칙은 AI가 "알아서 판단해서" 지키는 것들이었어요. Hook은 다르게, 특정 상황에서 무조건 자동으로 실행되는 안전장치예요. 예를 들어 비밀번호 같은 민감한 정보를 실수로 코드에 넣으려는 시도를 사람이 미처 못 챙겨도 자동으로 막아줄 수 있어요.
이렇게 부탁해보세요 비밀 키(.env 파일이나 API 키 같은 것)를 실수로 코드 파일에 쓰려고 하면 자동으로 막아주는 규칙(Hook)을 이 프로젝트에 만들어줘.
✅ 확인해보세요
Hook이 프로젝트 설정에 추가됐다고 AI가 답했나요?
6

데이터베이스에 연결해서, 새로고침해도 안 사라지게

하네스 · 맥락 확장
왜 이렇게 할까요? 지금까지 만든 사이트는 새로고침하면 처음 상태로 돌아가요. 실제 서비스들은 Supabase 같은 무료 데이터베이스에 내용을 저장해서, 어떤 기기로 접속해도 내가 넣은 내용이 그대로 남아있게 만들어요.

사람이 직접 할 일 — 3가지

  • supabase.com 가입 → New Project로 무료 프로젝트 생성
  • Project URL과 anon(공개) key를 메모장에 복사해두기
  • service(비밀) key는 따로 복사하지 않기 — 이건 코드에 절대 넣으면 안 되는 마스터 키예요
Claude Code에 Supabase를 MCP로 연결해달라고 부탁하세요 Supabase를 MCP로 연결해줘. 내 Project URL과 anon key는 이거야 — [여기에 붙여넣기]. 연결되면 알려줘.
연결되면, 테이블을 만들고 데이터를 채워달라고 하세요
왜 MCP는 매번 승인을 물을까요? 데이터베이스는 한 번 잘못 건드리면 되돌리기 어려워요. 그래서 Claude가 DB에 손대는 작업 전에는 매번 사람에게 확인을 받도록 되어 있어요 — 안전을 우선하는 연결 방식이에요.
✅ 확인해보세요
사이트를 새로고침해도 데이터가 그대로 남아있나요?
STEP 7 · 모두 함께

Vercel CLI로, 진짜 인터넷 주소가 있는 사이트로

Netlify Drop처럼 파일을 드래그하는 방법도 있지만, 오늘은 진짜 개발자들이 쓰는 CLI 배포를 해볼게요. Claude Code에게 그대로 맡기면 돼요.

💬
Claude Code에게 배포를 맡기세요아래 프롬프트 하나면 Vercel CLI 설치부터 로그인, 배포까지 다 알아서 진행해요.
🔑
중간에 로그인 창이 뜨면브라우저에서 이메일/구글 등으로 로그인 (처음이면 무료 가입). Claude가 권한을 물으면 안전한 요청은 Yes로 승인하세요.
⏳
몇 분 정도 걸릴 수 있어요Node와 Vercel CLI를 설치하는 시간이에요. 정상적인 과정이니 기다려주세요.
🔗
URL을 받으면 완성https://...vercel.app 형태의 주소가 나와요. 다른 기기로 열어서 사이트(+데이터)가 그대로 뜨면 성공!
Claude Code · 입력 이 사이트를 Vercel로 인터넷에 배포해줘. Vercel CLI가 없으면 네가 알아서 설치하고(전역에 한 번만), GitHub 없이 바로 배포해줘. 배포가 끝나면 접속 주소(URL)를 알려줘.

💡 CLI(Vercel)와 MCP(Supabase), 뭐가 다를까

  • CLI(Vercel) — 한 번 로그인해두면 이후로는 자동. 배포는 통째로 끝나는 단순 작업이라 위험도가 낮아서 속도를 우선해요.
  • MCP(Supabase) — 매 작업마다 승인. 데이터는 되돌리기 어려우니 보안을 우선해요.
⚠️ 잠깐, 이것만은 꼭 기억하세요.
AI가 코드를 대신 짜준다고 해서, 여러분이 "이해 안 해도 되는 것"은 아니에요. AI가 만든 결과를 보면서 "왜 이렇게 됐지?"를 스스로 생각해보는 습관이 훨씬 더 큰 힘이 돼요. 오늘 배운 건 코드 그 자체가 아니라, AI가 잘 일하는 환경을 설계하는 방법이에요.
